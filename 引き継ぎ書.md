## 1. システム概要

### 1-1. サイトの目的と構成

本ウェブサイトは神戸大学医学部附属病院リハビリテーション科の公式サイトです。
患者向け情報・スタッフ紹介・研究情報・後期研修案内を提供しています。

**技術スタック：**

| 項目 | 内容 |
|------|------|
| 方式 | 静的HTMLサイト（バックエンドサーバー不使用） |
| 言語 | HTML5 / CSS3 / JavaScript（バニラJS） |
| CMS | RehabCMS（独自開発・ブラウザ内完結） |
| フォント | Google Fonts（Noto Sans JP） |
| データ保存 | ブラウザのlocalStorage（サーバーDB不使用） |
| 外部フォーム | Google Forms（後期研修申込） |

### 1-2. ページ一覧

| ファイル名 | ページ名 | 内容 |
|-----------|---------|------|
| `index.html` | トップページ | バナー・お知らせ・診療科紹介 |
| `greeting.html` | 診療科長ご挨拶 | 科長メッセージ・写真 |
| `staff.html` | スタッフ紹介 | 医師・コメディカル・大学院生 |
| `info.html` | 患者さんへ | 外来スケジュール・対象疾患・アクセス |
| `study.html` | 研究・業績 | 研究テーマ・論文一覧・受賞歴 |
| `training.html` | 後期研修プログラム | 研修内容・応募フォーム |
| `style.css` | スタイルシート | 共通デザイン |
| `cms-engine.js` | CMSエンジン | 編集機能の本体 |

## 3. 日常の更新作業（CMS操作手順）

### 3-1. 編集モードの起動方法

**方法A：URLパラメーターを使う**
```
https://（サイトURL）/index.html?edit=1
```

**方法B：ボタンから起動する**
1. 通常表示でページ右下の「✏️ 編集モード」ボタンをクリック
2. パスワードを入力して認証

### 3-2. 初回パスワード設定

1. 編集モードを起動するとパスワード設定ダイアログが表示される
2. 6文字以上のパスワードを設定する
3. 確認のため同じパスワードを再入力する
4. 設定されたパスワードのハッシュ値がブラウザのlocalStorageに保存される

> **重要：** パスワードはシステムに保存されません（ハッシュ値のみ）。必ずメモに控えてください。

### 3-3. コンテンツの編集手順

1. 編集モードに入ると、編集可能な箇所がハイライト表示される
2. 編集したい箇所をクリックしてテキストを入力
3. テキストツールバー（太字・斜体・下線・リンク）が使用可能
4. 画像エリアをクリックするとファイル選択ダイアログが開く

### 3-4. お知らせ（ニュース）の更新

1. 編集モードでトップページのお知らせ欄に「＋ 行を追加」ボタンが表示される
2. ボタンをクリックで新しい行が追加される（日付は自動入力）
3. 日付・テキストを編集する
4. 不要な行は「✕」ボタンで削除する

### 3-5. 保存と公開の手順

**ステップ1：ブラウザに保存**
- ツールバーの「💾 保存」ボタンをクリック
- 「保存しました」のトーストが表示されれば成功

**ステップ2：JSONファイルに書き出し**
- ツールバーの「📥 JSONで書き出し」ボタンをクリック
- ファイル名：`rehab_[ページ名]_[YYYY-MM-DD].json`

**ステップ3：サーバーに反映**
- 書き出したJSONファイルを適切な場所に配置（サーバー構成による）
- または、HTMLファイルを直接更新してサーバーにアップロード

> **注意：** 保存はブラウザのlocalStorageに行われます。別のPC・ブラウザでは編集内容が引き継がれません。JSONファイルへの書き出しを必ず行ってください。

### 3-6. セッションについて

- 認証後のセッションは **4時間** で自動失効します
- ブラウザを閉じると即時失効します（sessionStorageを使用）
- 「ログアウト」ボタンで手動ログアウトも可能です

## 4. やってはいけないこと（セキュリティ・リスク管理）

### 4-1. 【最重要】パスワード・認証関連

```
禁止事項
```

| # | 禁止行為 | 理由・リスク |
|---|---------|------------|
| 1 | パスワードを他者と共有・メールで送付する | 不正アクセスの原因になる |
| 2 | 簡単なパスワード（`password123` 等）を使う | ブルートフォース攻撃で突破される |
| 3 | 公共PC・共有PCで編集モードにログインしたまま離席する | 第三者が不正に編集できる |
| 4 | 編集後にログアウトせずブラウザを閉じる（公共PCの場合） | sessionStorageが残る可能性がある |
| 5 | パスワードをソースコードやドキュメントにベタ書きする | リポジトリ流出時に漏洩する |

### 4-2. コンテンツ・情報管理

| # | 禁止行為 | 理由・リスク |
|---|---------|------------|
| 6 | 患者個人情報（氏名・生年月日・病歴等）をページに記載する | 個人情報保護法違反・プライバシー侵害 |
| 7 | スタッフの個人的な連絡先（携帯番号・私用メール）を掲載する | プライバシー侵害・ハラスメントリスク |
| 8 | 許可を取っていない写真・文章を掲載する | 著作権侵害 |
| 9 | 大学・病院の承認なしに公式見解として情報を掲載する | 組織としての誤解を招く可能性がある |
| 10 | 未確認の研究データや論文情報を掲載する | 研究倫理上の問題になる |

### 4-3. 技術・システム面

| # | 禁止行為 | 理由・リスク |
|---|---------|------------|
| 11 | `cms-engine.js` を無闇に改変する | 認証機能が壊れ、CMS全体が動作不能になる |
| 12 | `style.css` の `z-index` や認証関連CSSを削除する | 編集ダイアログが表示されなくなる |
| 13 | ブラウザの開発者ツールを使ってlocalStorageのハッシュを書き換える | セキュリティを意図的に破壊することになる |
| 14 | `data-cms` 属性を全削除する | 全ページの編集機能が失われる |
| 15 | `.DS_Store` ファイルをサーバーにアップロードする | サイト内部構造が外部に露出する可能性がある |
| 16 | HTTP（非暗号化）でサイトを公開したまま運用する | 通信が盗聴可能になる（必ずHTTPS化すること） |
| 17 | 外部の信頼できないJavaScriptを `<script>` タグで追加する | XSS・悪意あるコード実行のリスク |

### 4-4. データ管理

| # | 禁止行為 | 理由・リスク |
|---|---------|------------|
| 18 | localStorageのクリア（ブラウザデータ削除）を安易に行う | 保存されたコンテンツがすべて失われる |
| 19 | JSONバックアップを作成せずにlocalStorageを唯一のデータとして運用する | PC故障・ブラウザ更新で全データ消失 |
| 20 | 大容量画像（数MB超）を多数アップロードする | localStorageの容量上限（5〜10MB）を超えて保存エラーになる |

### 4-5. 運用ポリシー

| # | 禁止行為 | 理由・リスク |
|---|---------|------------|
| 21 | 誤字・不正確情報を確認なしに公開する | 患者・研修医への誤案内につながる |
| 22 | 外来スケジュール変更を事前に掲載せず当日更新する | 患者への混乱を招く |
| 23 | 担当者不在時に誰でも更新できる状態にする | 品質・正確性が担保されない |

### 5-2. パスワードのリセット方法

パスワードを忘れた場合は以下の手順でリセットします。

1. ブラウザの開発者ツールを開く（F12 または右クリック→「検証」）
2. 「Application」タブ →「Local Storage」→ サイトのURLを選択
3. `rehab_cms_pw_hash` のキーを見つけて削除
4. ページをリロードすると初回パスワード設定画面が表示される

> **注意：** この操作は管理者のみが行ってください。

### 5-3. バックアップ手順

**【推奨】JSONバックアップ（最低月1回）**
1. 各ページで編集モードに入る（認証は不要、閲覧のみでも可）
2. ツールバー →「📥 JSONで書き出し」をクリック
3. ダウンロードされたJSONファイルを日付フォルダに保存

```
backup/
├── 2026-02/
│   ├── rehab_index_2026-02-25.json
│   ├── rehab_greeting_2026-02-25.json
│   ├── rehab_staff_2026-02-25.json
│   ├── rehab_info_2026-02-25.json
│   ├── rehab_study_2026-02-25.json
│   └── rehab_training_2026-02-25.json
```

**HTMLファイル丸ごとバックアップ（3ヶ月ごと）**
- `files/` フォルダ全体をZIP圧縮して保存

### 5-5. 画像管理の注意事項

- アップロード画像はData URL（Base64）としてlocalStorageに保存されます
- 画像ファイルは **1枚あたり500KB以下** を推奨します（圧縮して使用）

## 6. 障害・トラブル対応

### 6-1. トラブルシューティング

| 症状 | 原因 | 対処法 |
|------|------|-------|
| 編集モードに入れない | パスワード忘れ | 5-2のパスワードリセット手順を実行 |
| 保存してもページに反映されない | 別のPC・ブラウザで開いている | 同じブラウザで確認。または再度JSONを書き出してアップロード |
| 保存エラーが出る | localStorage容量オーバー | 不要な画像を削除、または古いデータをクリアしてJSONから復元 |
| Googleフォームが表示されない | フォームの公開設定が変更された | Google フォームの管理者に連絡して公開設定を確認 |
| フォントが表示されない | Google Fontsへの接続不可 | ネットワーク確認。一時的な問題の場合は待機。完全オフライン環境の場合はフォントのローカル配信を検討 |
| ページが崩れる | CSSファイルの破損・欠損 | バックアップの `style.css` で上書き |
| CMSが動作しない | `cms-engine.js` のエラー | ブラウザのコンソール（F12）でエラーを確認。バックアップで復元 |

### 6-2. 緊急時の対応フロー

```
【サイトが表示されない・大きく崩れた場合】

1. バックアップのHTMLファイルで状態確認
   ↓
2. バックアップファイルを元に復元
   ↓
3. 復元後、JSONファイルからデータを再適用
   ↓
4. 動作確認後、サーバーに再アップロード
```

## 7. 次期サイト刷新時の引き継ぎ事項

### 7-1. 現システムの課題（刷新理由）

現システムは「ブラウザのlocalStorageに保存する静的サイト」という方式のため、
以下の制約・リスクがあります。次期システムでは改善が必要です。

| 課題 | 詳細 | 優先度 |
|------|------|-------|
| データがブラウザ依存 | 別PCで開いても編集内容が見えない | 高 |
| バックアップが手動 | 忘れるとデータ消失リスクがある | 高 |
| パスワードに塩（Salt）なし | SHA-256のみでレインボーテーブル攻撃に脆弱 | 高 |
| ログイン試行制限なし | ブルートフォース攻撃を防ぐ仕組みがない | 高 |
| 権限管理が一元 | 編集者ごとの権限設定ができない（全部か無か） | 中 |
| パスワードリセット機能なし | サーバーサイドのリセット機能がない | 中 |
| HTTPS強制なし | コード上でHTTPS要件が担保されていない | 中 |
| 画像管理が煩雑 | Data URLでの保存は肥大化しやすい | 中 |
| 複数人同時編集不可 | 競合が発生しても検知できない | 低 |

### 7-2. 次期システムへの移行チェックリスト

**コンテンツ移行**
- [ ] 各ページのテキストコンテンツを書き出してCSV・Excelで整理する
- [ ] 画像ファイルを全て書き出して適切な名前で保存する
- [ ] お知らせ（ニュース）一覧をスプレッドシートに整理する
- [ ] スタッフ情報（氏名・役職・肩書）を一覧化する
- [ ] 外来スケジュール表を画像またはCSVで保存する
- [ ] 論文・業績リストを整理する
- [ ] Google Formsのリンクを確認・引き継ぐ

**デザイン・UI要件**
- [ ] 現サイトのカラーコードを記録する（#1a5fa8, #1a2f6e 等）
- [ ] フォント（Noto Sans JP）の使用を継続するか決定する
- [ ] スマートフォン対応（レスポンシブ）の要件を確認する
- [ ] ナビゲーション構成（ページ数・ページ名）を確定する
- [ ] アクセシビリティ要件（WCAG対応レベル）を確認する

**機能要件（次期システムで必要な機能）**
- [ ] ブラウザ・PC間でコンテンツが共有されるCMS
- [ ] 複数ユーザーの権限管理（閲覧専用・編集・管理者）
- [ ] 変更履歴・バージョン管理
- [ ] 定期自動バックアップ
- [ ] 画像のサーバー側管理
- [ ] パスワードリセット機能（メール通知）
- [ ] HTTPS必須

**セキュリティ要件**
- [ ] パスワードのハッシュ化（bcrypt または Argon2 推奨）
- [ ] ログイン試行制限（5回失敗でロックアウト等）
- [ ] 管理画面へのアクセス制限（IPアドレス制限等）
- [ ] 入力値のサニタイズ・バリデーション
- [ ] CSRF対策
- [ ] セキュリティヘッダーの設定

**インフラ・運用**
- [ ] ホスティング先の選定（大学サーバー・クラウド）
- [ ] ドメイン・SSL証明書の管理移行
- [ ] 旧URLのリダイレクト設定
- [ ] 新システムの操作マニュアル作成
- [ ] 管理者・編集者への操作研修

### 7-3. 推奨する次期システムの方向性

**軽量な改善（コスト低）：**
- 現在のHTMLサイトをベースに、[Netlify CMS](https://decapcms.org/) や [TinaCMS](https://tina.io/) を追加
- Gitベースのコンテンツ管理が可能になり、バックアップ・履歴管理が解決される

**中規模な刷新（コスト中）：**
- WordPress や Craft CMS 等の実績あるCMSへの移行
- 大学内のシステム管理部署との連携が必要

**本格的なリニューアル（コスト高）：**
- Next.js / Astro 等モダンフレームワーク + ヘッドレスCMS
- 高いパフォーマンスとセキュリティ、柔軟な拡張性

### 7-4. 移行時に保持すべきURL（SEO・ブックマーク対策）

現在のページURLが変わる場合は、旧URLから新URLへのリダイレクトを設定すること。

```
/index.html       → /
/greeting.html    → /greeting/ または /about/
/staff.html       → /staff/ または /team/
/info.html        → /information/ または /patients/
/study.html       → /research/ または /publications/
/training.html    → /training/ または /residency/
```
